
# 📋 Повний перелік команд з діапазонами

Ось всі доступні команди для керування осцилографом через USB-CDC термінал:

---

## 🔹 Основні команди

| Команда | Параметри | Діапазон | Опис |
|---------|-----------|----------|------|
| `CAPTURE:START` | — | — | ▶️ Увімкнути передачу даних |
| `CAPTURE:STOP` | — | — | ⏹ Зупинити передачу |
| `Test signal:0` | `0` або `1` | `0` = вимк, `1` = увімк | 🌀 Увімкнути/вимкнути тестові хвилі |
| `HELP` або `?` | — | — | ❓ Показати список команд |
| `STATUS` | — | — | 📊 Показати поточний стан системи |

---

## 🔹 Керування частотою семплінгу

| Команда | Параметр | Діапазон | Опис |
|---------|----------|----------|------|
| `Rate:XX` | `1..1000` (мс) | `1` = 1 мс, `1000` = 1 с | 🔄 Період між пакетами. Частота АЦП = `(1000/Rate) × 4` |
| `Freq:XXXX` | `100..50000` (Гц) | `100` = 100 Гц, `50000` = 50 кГц | ⚡ Пряме задання частоти семплінгу АЦП |

### 📊 Приклади для `Rate:`:
| Команда | Період | Частота пакетів | Частота АЦП (4 канали) |
|---------|--------|-----------------|------------------------|
| `Rate:1` | 1 мс | 1000 пакетів/с | **4000 Гц** |
| `Rate:5` | 5 мс | 200 пакетів/с | **800 Гц** |
| `Rate:10` | 10 мс | 100 пакетів/с | **400 Гц** |
| `Rate:20` | 20 мс | 50 пакетів/с | **200 Гц** |
| `Rate:100` | 100 мс | 10 пакетів/с | **40 Гц** |
| `Rate:1000` | 1 с | 1 пакет/с | **4 Гц** |

### 📊 Приклади для `Freq:`:
| Команда | Частота АЦП | Еквівалентний період |
|---------|-------------|---------------------|
| `Freq:50000` | 50 кГц (макс) | ~0.08 мс |
| `Freq:10000` | 10 кГц | 0.4 мс |
| `Freq:1000` | 1 кГц | 4 мс |
| `Freq:400` | 400 Гц | 10 мс |
| `Freq:100` | 100 Гц (мін) | 40 мс |

> 💡 **Порада:** Використовуйте `Rate:` для зручного налаштування в мс, або `Freq:` для точного задання в Гц.

---

## 🔹 Налаштування тригера

| Команда | Параметр | Діапазон | Опис |
|---------|----------|----------|------|
| `TRIG:LEVEL:X.XX` | `-1.0..1.0` | `-1.0` = мінімум, `1.0` = максимум | 🎯 Рівень спрацьовування тригера |
| `TRIG:EDGE:rise` | `rise` / `fall` / `auto` | — | ⬆️ Спрацьовування по фронту |
| `TRIG:EDGE:fall` | — | — | ⬇️ Спрацьовування по спаду |
| `TRIG:EDGE:auto` | — | — | 🔄 Автоматичний режим |
| `TRIG:HYST:X.XX` | `0.01..0.5` | `0.01` = 1%, `0.5` = 50% | 📐 Гістерезис тригера (відсотки) |

### Приклади:
```bash
TRIG:LEVEL:0.5      # Тригер на рівні 50% від амплітуди
TRIG:EDGE:rise      # Спрацьовування при зростанні сигналу
TRIG:HYST:0.1       # Гістерезис 10% (захист від дребезгу)
```

---

## 🔹 Налаштування каналів (CH1..CH4)

| Команда | Параметр | Діапазон | Опис |
|---------|----------|----------|------|
| `CH1:ON` / `CH1:OFF` | `ON` / `OFF` (або `1` / `0`) | — | 🔌 Увімкнути/вимкнути канал 1 |
| `CH1:SCALE:X.XX` | `0.2..2.2` | `1.0` = норма, `2.0` = ×2 збільшення | 📏 Масштабування амплітуди каналу |
| `CH1:OFFSET:XXX` | `-250..250` | `0` = без зсуву | ↔️ Вертикальний зсув каналу |

> 🔄 Замініть `CH1` на `CH2`, `CH3` або `CH4` для інших каналів.

### Приклади:
```bash
CH1:ON                # Увімкнути канал 1
CH2:SCALE:1.5         # Збільшити амплітуду каналу 2 в 1.5 рази
CH3:OFFSET:-50        # Зсунути канал 3 вниз на 50 одиниць
CH4:OFF               # Вимкнути канал 4
```

## 🔹 Налаштування часової розгортки

| Команда | Параметр | Допустимі значення | Опис |
|---------|----------|-------------------|------|
| `TIME/DIV:XX` | `1/2/5/10/20/50/100` | Тільки ці числа | ⏱ Час на поділ сітки (в мс) |

### Приклади:
```bash
TIME/DIV:1    # 1 мс/поділ — для швидких сигналів
TIME/DIV:10   # 10 мс/поділ — стандартний режим
TIME/DIV:100  # 100 мс/поділ — для повільних процесів
```

## 📡 Приклади сесій терміналу

### 🔹 Швидкі сигнали (аудіо, вібрація):
```bash
Freq:8000           # 8 кГц семплінг
CAPTURE:START       # Почати передачу
```

### 🔹 Повільні сигнали (температура, потенціометр):
```bash
Rate:200            # 200 мс між пакетами = 5 пакетів/с
CAPTURE:START
```

### 🔹 Налаштування тригера:
```bash
TRIG:LEVEL:0.3      # Тригер на 30% амплітуди
TRIG:EDGE:rise      # По фронту
TRIG:HYST:0.05      # Гістерезис 5%
CAPTURE:START
```

### 🔹 Перевірка стану:
```bash
STATUS
# Відповідь:
# >STATUS: USB=1 CAP=1 TS=0 ADC=8000Hz PktRate=2000/s (0ms)
```

---

## ⚠️ Важливі зауваження

1. **Регістр не має значення** для ключових слів: `rate:10` = `Rate:10` ✅
2. **Пробіли після `:` допускаються**: `Rate: 10` = `Rate:10` ✅
3. **Команди виконуються миттєво** — не потрібно перезавантажувати плату.
4. **Якщо параметр поза діапазоном** — отримаєте `>ERR: ...` і значення не зміниться.
5. **Для застосування змін частоти** не потрібно зупиняти `CAPTURE` — зміни застосовуються "на льоту".

---

## 🛠 Діагностика проблем

| Симптом | Можлива причина | Рішення |
|---------|----------------|---------|
| `>ERR: Rate must be 1..1000 ms` | Параметр поза межами | Перевірте число: `Rate:50` |
| `>ERR: Unknown command 'xxx'` | Одруківка в команді | Перевірте написання: `HELP` |
| Немає відповіді на команди | Термінал не надсилає `\n` | Налаштуйте закінчення рядка на `LF` або `CRLF` |
| Дані йдуть, але графік "скаче" | Неправильний `ADC_CENTER` | Перевірте `config.h`: `#define ADC_CENTER (-2048)` |

```bash
Rate:10

Freq:4000

# 🎯 Тригер
TRIG:LEVEL:0.0

CH1:ON
CH2:ON
CH3:OFF
CH4:OFF
CH1:SCALE:1.0
CH1:OFFSET:0

# ⏱ Часова розгортка
TIME/DIV:10

# 📊 Перевірка
STATUS

# ⏹ Стоп
CAPTURE:STOP
```
